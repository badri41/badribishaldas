Computer Architecture & OS: A Reading List

A compact reading list for operating systems, architecture, and memory hierarchy. This is the background material that keeps making the C++ concurrency work feel less mysterious.

Table of Contents

Overview

I originally started this list because I wanted better context for concurrency in C++. It turned out to be useful well beyond that. Once caches, virtual memory, scheduling, and coherence protocols stop being background words, the systems picture becomes easier to reason about.

Operating Systems

  • Operating Systems: Three Easy Pieces is my primary OS resource right now.
  • Operating System Concepts.
  • Operating Systems: Principles and Practice.
  • OS Volume 1: Kernels and Processes.
  • OS Volume 2: Concurrency.
  • OS Volume 3: Memory Management.
  • OS Volume 4: Persistence.
  • Professor Sarangi's operating systems material.

Channels I keep revisiting

Core Dumped is still one of my favorite channels for OS visualizations.

Computer Architecture

  • Computer Architecture by Hennessy and Patterson.
  • Computer Organization and Design.
  • Computer Systems: A Programmer's Perspective.
  • Inside the Machine by Jon Stokes.
  • A Primer on Memory Consistency and Cache Coherence.
  • IIT Delhi architecture notes and memory hierarchy material.
