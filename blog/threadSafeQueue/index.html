ThreadSafeQueueLib: A Concurrent Queue Library in C++

That time I got reincarnated as a lock-free queue library implementer. A project write-up on design decisions, memory ordering, false sharing, and debugging concurrency.

Table of Contents

Overview

I am doing this project under the mentorship and guidance of Toshit Bhaiya. This project is in work-in-progress.

Github

Motivation

A standard queue interface becomes fragile under contention. Something as ordinary as checking empty() before front() and pop() is prone to data races, and hence not suitable. so there is a need for a thread-safe queue that can handle concurrent access without data races and provide suitable latency and throughput under contention.

Design Decisions

Why lock-free?

Mutex-based queues are much easier to write and reason about. The lock-free route only makes sense if the learning goal is precisely to understand progress guarantees, contention, and the parts of the design where performance meets correctness.

Bounded versus unbounded

Bounded queues are attractive because the memory is fixed and predictable. Unbounded queues avoid the full-buffer problem but push you toward allocation and reclamation concerns that are easy to underestimate.

Compile-time configuration

I plan to use templates for compile-time configuration, allowing the compiler to optimize based on known parameters at build time.

The Interesting Parts

Memory ordering

The C++ memory model is the key pillarstone behind understanding concurrency and atomics. Using seq_cst everywhere is safe but slow af. The real work is deciding where acquire-release is enough and where relaxed operations are still correct.

False sharing

Producer and consumer indices sharing a cache line can quietly destroy throughput through invalidation traffic. I didn't knew the reason why? so I had to read about cache line coherence, the MESI protocol and how to use padding and alignment to mitigate false sharing. (More about it in a future post on systems topics.)

Debugging concurrency

It is not trivial to prove that your code is comepletely free of data races. Tools like ThreadSanitizer are helpful but have their limitations.

Behind the Project

The project also pushed me back into systems reading. I needed stronger OS and architecture knowledge to reason cleanly about what the queue was doing under the hood.

What Is Next

I am currently implementing the SPSC unbounded version of the queue. After that, I will move on to the MPMC Lockfree unbounded version.
