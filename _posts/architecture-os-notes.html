Computer Architecture & OS: Where My Brain Meets the Memory Wall

Computer Systems / OS / Performance New

How this started

I started with "let me quickly learn concurrency" and somehow ended up reading about caches, TLBs, virtual memory, scheduling, and why my CPU is doing black magic behind my back.

Now when code runs fast, I am happy. When code runs slow, I blame cache misses first and myself second.

Main learning path

My favorite architecture course is the CMU playlist by OmurLectures, delivered by Professor Onur Mutlu in Carnegie Mellon University's Undergraduate Computer Architecture class (18-447), Spring 2015:

Where this connects to C++

Learning OS and architecture makes C++ concurrency less mystical. Suddenly memory_order_acquire is not a random ritual, and atomics stop looking like ancient runes.

I still get confused, but now the confusion is high quality.

Current mood

Half student, half detective, full-time cache-miss victim. But this is genuinely one of the most fun learning tracks I have done.
